# Visual

## Descripción

Requisitos de presentación visual: contraste de color, tipografía legible, uso de iconos y gráficos, y no depender solo del color para transmitir información. Incluye consideraciones para zoom y redimensionamiento de texto.

## Alcance

Afecta a interfaces gráficas (UI), documentos y contenido multimedia con componente visual. No sustituye el trabajo de contraste en código (desarrollo) ni de alternativas textuales (contenido).

## Checklist

- Contraste mínimo 4,5:1 para texto normal y 3:1 para texto grande (AA).
- Texto redimensionable hasta 200% sin pérdida de función.
- Iconos y estados no comunicados únicamente por color.
- Espaciado y tamaños tocables/clickeables suficientes (mín. 44×44 px recomendado).

## Errores comunes

- Usar grises claros sobre blanco para texto body.
- Botones o enlaces que solo se distinguen por color.
- Iconos sin etiqueta o contexto (p. ej. solo una lupa para “buscar”).

## Cómo validar

- Herramientas de contraste (p. ej. Contrast Checker, DevTools).
- Zoom al 200% y comprobación de layout y legibilidad.
- Revisión con simulaciones de daltonismo (herramientas o filtros).
